# サイトが開かない場合のトラブルシューティング

## 1. ローカルで動作確認

まず、ローカルでサイトが動作するか確認してください。

```powershell
cd C:\Users\motim\.cursor\study-site
npm run dev
```

ブラウザで `http://localhost:5173` を開いて、サイトが表示されるか確認してください。

## 2. Vercelのデプロイログを確認

1. [Vercelダッシュボード](https://vercel.com/dashboard) にログイン
2. プロジェクトを選択
3. **Deployments** タブを開く
4. 最新のデプロイをクリック
5. **Build Logs** を確認
   - エラーがあれば、その内容をメモしてください

## 3. ブラウザのコンソールを確認

1. デプロイされたサイトを開く
2. **F12** で開発者ツールを開く
3. **Console** タブを確認
   - 赤いエラーメッセージがあれば、その内容をメモしてください
4. **Network** タブを確認
   - 404エラー（赤い表示）がないか確認

## 4. よくある問題と解決方法

### 問題: 真っ白な画面
**原因**: JavaScriptの読み込みエラー
**解決**: ブラウザのコンソールでエラーを確認

### 問題: 404エラー
**原因**: アセット（JS/CSS）のパスが間違っている
**解決**: `vite.config.ts`の設定を確認

### 問題: ビルドエラー
**原因**: 依存パッケージの問題
**解決**: `package.json`を確認

## 5. 再デプロイ

設定を修正したら、GitHubにプッシュして再デプロイ：

```powershell
git add .
git commit -m "設定を修正"
git push
```

Vercelが自動で再デプロイします。
